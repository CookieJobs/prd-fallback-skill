# prd-fallback

一个 Claude Code 技能（Skill），帮助产品经理在交付 PRD 之前，自动识别缺失的兜底逻辑和边界条件，并在飞书文档中直接插入带方案选项的高亮建议块。

## 为什么需要它

PRD 中最容易遗漏的不是主流程，而是各种异常场景的处理方式——网络失败怎么办、输入了奇怪的字符怎么办、业务规则不满足怎么办。这些"兜底逻辑"如果没有写进 PRD，就会在研发阶段变成口头约定，在测试阶段变成争议点，在上线后变成 Bug。

这个技能让 AI 扮演"资深产品安全审查员"，系统性地扫描 PRD 并给出建议，PM 只需在飞书文档里选择方案就能完成补全。

## 效果预览

运行后，AI 先展示终端预览摘要：

```
📋 PRD 审查完成，发现 7 处需要补充兜底逻辑：

 #  位置                    类型          场景摘要
──────────────────────────────────────────────────────
 1  3.2 提交订单            🔴 业务风险   库存校验通过后超卖
 2  3.2 提交订单            🟡 网络错误   提交请求失败/超时
 3  3.4 上传凭证            🟡 输入校验   文件格式/大小边界
 ...

输入要跳过的编号（如：skip 3,7），或直接按回车全部插入：
```

确认后，AI 在飞书 PRD 文档中对应位置插入彩色 callout 块：

- 🔴 **红色**：高风险场景（数据丢失、资金异常），必须处理
- 🟡 **黄色**：常规异常（网络错误、输入校验），建议处理
- 💡 **绿色**：体验优化建议，可选

每个 callout 块包含一个稳定的短哈希 ID（如 `FB-a3f7`），PM 可以用 ID 与 AI 继续对话精修某个建议。

## 覆盖的边界条件类型

- **网络/系统错误**：请求失败、超时、5xx
- **业务逻辑失败**：库存不足、余额不够、权限不足、重复提交
- **输入校验边界**：emoji、特殊字符、超长文本、纯空格
- **空状态/加载态**：列表为空、首次使用、加载超时
- **权限/登录态**：会话过期、无权限、多端冲突
- **并发/竞态**：快速双击、多设备同时操作

不局限于以上类别——AI 会根据具体功能场景自由判断其他边界情况。

## 安装

**前置条件**：

- [Claude Code](https://claude.ai/code) 已安装
- [lark-cli](https://github.com/larksuite/lark-cli) 已安装并完成飞书授权
- `lark-doc` 等飞书相关技能已配置（用于文档读写）

**安装步骤**：

```bash
# 将技能文件夹复制到 Claude Code 技能目录
cp -r prd-fallback ~/.claude/skills/

# 重启 Claude Code，技能自动加载
```

验证安装：在 Claude Code 中运行 `/skills`，看到 `prd-fallback` 即为成功。

## 使用方式

**初次审查 PRD**：

```
/prd-fallback https://xxx.feishu.cn/docx/xxx
```

或者自然语言触发：

```
帮我审查这份 PRD 的兜底逻辑 https://xxx.feishu.cn/docx/xxx
```

**精修某个建议块**（Path B）：

```
FB-a3f7 的方案 B 太技术化了，帮我改成更偏用户感知的描述
```

AI 会通过 ID 找到飞书文档中对应的 callout 块，直接修改内容，其他内容不受影响。

## 工作流程

```
PM 提供飞书 URL
    ↓
AI 读取 PRD 全文（>8000字时自动分段）
    ↓
AI 分析边界条件缺口
    ↓
终端展示预览摘要（PM 可 skip 不需要的条目）
    ↓
PM 确认
    ↓
AI 生成带 FB-xxxx ID 的 callout 块，批量写入飞书
    ↓
PM 在飞书文档中选择方案，删除其余选项
```

## 注意事项

- 技能需要对飞书文档有**编辑权限**（读写权限）才能插入 callout 块
- PRD 使用表格结构时，callout 块会插入在对应表格行内（上下文清晰）
- 已有明确兜底描述的位置会自动跳过，不重复建议
- 每次运行前会检查文档中已有的 `FB-` ID，避免重复插入

## 迭代方向

- 支持 PM 在预览阶段修改 AI 判断的严重等级
- 支持批量标记「已处理」（将 callout 块替换为普通文本记录决策）
- 支持分析多份关联 PRD 的跨文档一致性
